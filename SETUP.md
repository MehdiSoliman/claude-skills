# Setup Guide

How to install the skill library and configure a project to use it.

---

## 1. Install the Skill Library

Clone this repository to your machine. We recommend keeping it in `~/Documents/`:

```bash
git clone https://github.com/MehdiSoliman/claude-skills.git ~/Documents/claude-skills
```

---

## 2. Register the Marketplace with Claude Code

Tell Claude Code where to find the skill library by adding the marketplace path to your global Claude settings.

Open (or create) `~/.claude/settings.json` and add:

```json
{
  "skillMarketplaces": [
    "~/Documents/claude-skills"
  ]
}
```

> If you cloned to a different path, use that path instead.

That's it — all skills are now available in every Claude Code session on your machine.

---

## 3. Configure a Project

For each project where you want to use skills, add two things at the project root:

### `_project/config.yaml`

Copy from [`_templates/project-config.yaml`](./_templates/project-config.yaml) and fill in your values:

```yaml
user_name: "Your Name"
project_name: "My Project"
communication_language: "English"       # language Claude speaks to you
document_output_language: "English"     # language for output documents
planning_artifacts: "planning"          # folder where PRDs are written
project_knowledge: "knowledge"          # folder where input docs live
```

### Folder structure

```
my-project/
├── _project/
│   └── config.yaml          ← your config (above)
├── planning/                ← created automatically on first run
└── knowledge/               ← put your input documents here
    ├── brief.md             ← product brief (optional but recommended)
    ├── research.md          ← research notes (optional)
    └── project-context.md  ← generated by analyze-codebase skill
```

> **Tip:** If your project already has a `CLAUDE.md` or `docs/` folder with good documentation, point `project_knowledge` at that folder instead of creating a new one.

---

## 4. Run a Skill

Open Claude Code in your project folder and use natural language to trigger a skill:

| What you say | Skill triggered |
|---|---|
| *"I want to create a new PRD"* | `create-prd` |
| *"Analyze my codebase"* | `analyze-codebase` |
| *"Check implementation readiness"* | `check-implementation-readiness` |

---

## 5. Keep Skills Up to Date

Pull the latest skills whenever the library is updated:

```bash
cd ~/Documents/claude-skills
git pull
```

---

## Recommended Workflow for a New Brownfield Project

```
1. analyze-codebase
   → produces knowledge/project-context.md

2. create-prd
   → detects project-context.md, starts in brownfield mode
   → produces planning/prd.md

3. check-implementation-readiness
   → validates planning/prd.md before development starts
```

For a greenfield project, skip step 1 and go straight to `create-prd`.
